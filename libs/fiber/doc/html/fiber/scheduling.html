Scheduling

Fibers are managed by a scheduler which coordinates the sequence of fibers running or waiting. Each thread is required to have its own scheduler. By default, Boost.Fiber creates for each thread a scheduler ( round_robin).

A fiber-scheduler must implement interface sched_algorithm. You are explicitly permitted to code your own sched_algorithm subclass, and to pass it to set_scheduling_algorithm(). 

void thread_fn() {
    my_fiber_scheduler mfs;
    boost::fibers::set_scheduling_algorithm( & mfs);
    ...
}

Class sched_algorithm 

#include <boost/fiber/algorithm.hpp>

struct sched_algorithm {
    virtual ~sched_algorithm() {}

    virtual void awakened( fiber_context *) = 0;

    virtual fiber_context * pick_next() = 0;
};

Member function awakened() 

virtual void awakened( fiber_context * f) = 0;

Effects:

Marks fiber f, to be ready to run.

Member function pick_next() 

virtual fiber_context * pick_next() = 0;

Effects:

Depending on the scheduling algorithm, this function returns the fiber which is to be resumed next.

Member function priority() 

virtual void priority( fiber_context *, int) noexcept = 0;

Effects:

Resets the priority of fiber f.

Class round_robin

This class implements sched_algorithm and schedules fibers in round-robin fashion (ignores fiber::priority()).
