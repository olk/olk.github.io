Scheduling

Fibers are managed by a scheduler which coordinates the sequence of fibers running or waiting. Each thread is required to have its own scheduler. By default, Boost.Fiber allocates for each thread a scheduler ( round_robin) on the heap.

Class round_robin

This class implements sched_algorithm and schedules fibers in round-robin fashion (ignores fiber::priority()).
