Notes to Windows

On Windows logical processors are identified by a Goup ID (up to 64 processors in one group) and a processor ID (represented by 64bit bitmask). The value for topo_t::processor_id is computed by group_id*64 + position of logical processor in the bitmask.

Scheduling

The decision to which logical processor a new job has to be scheduled has to be made by made by the code using Boost.Job. The application using Boost.Job has a better understanding of the job-graph (for instance which jobs depend on each other, on which physical memory bank is the processed data allocated etc.).
