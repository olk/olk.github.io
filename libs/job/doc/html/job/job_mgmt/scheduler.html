Constructor

scheduler( std::vector< topo_t > const& topology);

Effects:

Depending on parameter topology, worker-threads are created and pinned to the logical processors. The FiberPool strategy determines how worker-fibers are launched.

Note:

StackAllocator is required to allocate a stack used by the worker-fibers. If StackAllocator is not explicitly passed numa_fixedsize_stack is used by default. As default fiber pool static_pool<> with 64 worker-fibers is used.

Destructor 
~scheduler();

Effects:

Interrupts and joins the worker-fibers and -threads.

Throws:

Nothing.

submit_preempt( uint32_t cpuid, Fn && fn, Args && ... args);

Returns:

a std::future associated with the job.

Note:

The job is executed by the worker-thread running on logical processor wht the ID cpuid. If no Allocator is passed, numa_allocator<> is used. Calling std::future<>::get() inside a job is not adviced because it could block the worker-thread.
