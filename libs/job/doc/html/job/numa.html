Class numa_allocator

Boost.Job offers functionality to allocate physical memory on the memory banks of a NUMA note [5], to which a logical processor belongs. This feature can be used to reduce the latency of memory access, e.g. data is allocated on physical memory near the logical processor. For instance numa_fixedsize_stack allocates the stacks of the worker-fibers on the physical memory belonging to the NUMA node of the dedicated logical processor (the worker-thread is pinned to).


[5] NUMA == non-uniform memory access
